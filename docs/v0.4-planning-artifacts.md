# v0.4 Planning Artifacts

Trailmind v0.4 adds first-class management for spec and plan artifacts.

## Specs

A spec captures design decisions for a feature.

```sh
trailmind plan spec init \
  --epic projects/demo_app/mvp \
  --title "Parser Redesign" \
  --author alice@example.com
```

Specs are stored at `<epic>/docs/specs/YYYY-MM-DD-<slug>.md`.

### Spec Statuses

| Status | Meaning |
| --- | --- |
| `draft-for-review` | Initial draft, seeking review. |
| `approved-for-spec` | Spec approved; plan can be written. |
| `approved-for-implementation` | Spec and plan approved; implementation can begin. |
| `superseded` | Replaced by a newer spec. |

```sh
trailmind plan spec list --epic projects/demo_app/mvp
trailmind plan spec show projects/demo_app/mvp/docs/specs/2026-07-02-parser-redesign.md
trailmind plan spec set-status <ref> --status approved-for-spec --actor alice
```

## Plans

A plan describes how to implement an approved spec.

```sh
trailmind plan init \
  --epic projects/demo_app/mvp \
  --title "Parser Implementation" \
  --author alice@example.com \
  --spec docs/specs/2026-07-02-parser-redesign.md
```

Plans are stored at `<epic>/docs/plans/YYYY-MM-DD-<slug>.md`.

### Plan Statuses

| Status | Meaning |
| --- | --- |
| `draft` | Initial draft. |
| `approved` | Reviewed and approved for execution. |
| `in-progress` | Implementation underway. |
| `completed` | All generated tasks done. |
| `superseded` | Replaced by a newer plan. |

```sh
trailmind plan list --epic projects/demo_app/mvp
trailmind plan show projects/demo_app/mvp/docs/plans/2026-07-02-parser-implementation.md
trailmind plan set-status <ref> --status approved --actor alice
```

## Linking Specs and Plans

```sh
trailmind plan link-spec \
  --plan projects/demo_app/mvp/docs/plans/2026-07-02-parser-implementation.md \
  --spec projects/demo_app/mvp/docs/specs/2026-07-02-parser-redesign.md
```

## Plan Breakdown and Generated Tasks

When you run `trailmind plan breakdown --write`, the created task IDs are
automatically recorded in the source plan's `generated_tasks` frontmatter.
This provides reverse traceability from plan to tasks.

If the plan had no formal frontmatter (legacy plan), `plan breakdown --write`
adds minimal frontmatter with `title`, `created`, `status`, and `generated_tasks`.

See `docs/v0.4-plan-breakdown.md` for the breakdown command details.
