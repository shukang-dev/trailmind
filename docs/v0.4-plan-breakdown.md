# v0.4 Plan Breakdown

Trailmind v0.4 adds plan breakdown: a deterministic way to preview and create task drafts from an approved Markdown implementation plan.

## Preview

```sh
trailmind plan breakdown docs/plans/v0.4.md \
  --epic projects/demo_app/mvp \
  --filer alice@example.com \
  --owner alice@example.com
```

Preview mode is read-only. It reports which task sections would become Trailmind tasks and whether any source sections have already been generated.
`--filer` and `--owner` accept either roster emails or roster shortnames.

## Write

```sh
trailmind plan breakdown docs/plans/v0.4.md \
  --epic projects/demo_app/mvp \
  --filer alice@example.com \
  --owner alice@example.com \
  --write
```

Write mode creates ordinary task files under the target epic. Generated tasks include `source_plan`, `source_task`, and `source_heading` frontmatter so the original plan section remains traceable.

## JSON

```sh
trailmind plan breakdown docs/plans/v0.4.md \
  --epic projects/demo_app/mvp \
  --filer alice@example.com \
  --owner alice@example.com \
  --json
```

Use `--json` when another tool needs structured preview or write results.

## Duplicate Handling

Trailmind skips a generated task when another task in the same epic already has the same `source_plan` and `source_task`.

Use `--force` only when creating another task from the same plan section is intentional.

## Supported Plan Format

Plan breakdown supports implementation plans with level-three task headings:

```markdown
### Task 1: Parser Model

**Files:**
- Create: `src/trailmind/plan_breakdown.py`
- Test: `tests/test_plan_breakdown.py`

- [ ] **Step 1: Write the failing test**
```

It is not an AI planner and does not execute the plan.
