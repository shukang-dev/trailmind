# v0.3 Agent Handoff

Trailmind v0.3 adds pickup context packs for task and issue handoff. A pickup pack is a bounded, deterministic summary that helps a new human or AI agent understand what to do next.

## Task Pickup

```sh
trailmind task pickup T-123456-001
trailmind task pickup T-123456-001 --json
trailmind task pickup T-123456-001 --no-excerpts
trailmind task pickup T-123456-001 --log --actor alice
```

Task pickup includes the task status, owner, scope, acceptance notes, dependencies, deliverables, linked issues, recent Activity Log entries, bounded file excerpts, warnings, and next actions.

## Issue Pickup

```sh
trailmind issue pickup I-123456-001
trailmind issue pickup I-123456-001 --json
trailmind issue pickup I-123456-001 --no-excerpts
trailmind issue pickup I-123456-001 --log --actor alice
```

Issue pickup includes the issue status, severity, description, linked tasks, carry-forward metadata, recent Activity Log entries, bounded file excerpts from directly linked tasks, warnings, and next actions.

## Output Modes

Markdown is the default output and is designed for direct handoff into an agent session.

Use `--json` when another tool needs structured data.

## Bounded Context

Pickup only reads explicit file references such as task `code_paths` and `design_doc`. Each file excerpt is limited by `--max-lines`, which defaults to 80.

Use `--activity-limit` to control how many Activity Log entries appear. The default is 10.

Use `--no-excerpts` when paths are enough.

## Logging

Pickup is read-only by default. Add `--log --actor alice` to record an explicit handoff entry in the task or issue Activity Log.
